3 hướng triển khai Layout V3

Cùng design language (Dark sidebar navy #0f172a, topbar trắng, shadcn-vue) — khác nhau ở mức độ refactor

1

Nâng cấp tại chỗ (Incremental)

Giữ nguyên file layouts/admin.vue, chỉ thay CSS classes + thêm shadcn components vào từng phần. Sidebar đổi sang navy, nav items dùng shadcn Button variant ghost, topbar dùng shadcn DropdownMenu cho user menu.

Ưu điểm

  • Ít rủi ro, dễ review
  • Không phá vỡ pages hiện có
  • Nhanh nhất

Nhược điểm

  • Cấu trúc file vẫn monolith
  • Khó mở rộng sau này
2

Tách component (Recommended ✓)

Tách admin.vue thành 3 component riêng: AppSidebar.vue, AppTopbar.vue, AppDashboard.vue. Dùng shadcn Sheet cho mobile sidebar, DropdownMenu cho user, Tabs cho dashboard. Layout store giữ trạng thái sidebar.

Ưu điểm

  • Dễ maintain và test riêng từng phần
  • Reusable components
  • Phù hợp mở rộng module mới

Nhược điểm

  • Cần refactor admin.vue và home-admin.vue
3

Full Design System (Ambitious)

Xây dựng thư mục components/layout/ đầy đủ với Sidebar, Topbar, NavItem, SectionLabel, UserMenu. Thêm components/dashboard/ riêng với DashboardTabs, ReminderCard, ChartCard. Dùng shadcn ScrollArea, Separator, Avatar, Badge.

Ưu điểm

  • Chuẩn nhất, dễ scale
  • Từng component độc lập

Nhược điểm

  • Tốn công nhất
  • Overkill nếu team nhỏ
